# TaskFlow Portal

## Descrição do Projeto
TaskFlow Portal é uma aplicação web simples de gerenciamento de tarefas desenvolvida na plataforma Bubble. O sistema foi criado com o objetivo de oferecer uma solução prática para organização de atividades, permitindo que usuários realizem cadastro, login, acesso a um dashboard personalizado, criação de tarefas e visualização de sua própria lista de tarefas.

A proposta do projeto foi desenvolvida como parte de uma atividade acadêmica com foco em fundamentos de no-code/low-code, modelagem de dados, workflows, interface e publicação de aplicações web funcionais.

---

## Objetivo
O principal objetivo do TaskFlow Portal é resolver um problema real de organização e acompanhamento de tarefas por meio de uma aplicação funcional, acessível e construída sem programação tradicional extensiva.

---

## Público-Alvo
A aplicação é destinada a:
- estudantes;
- profissionais autônomos;
- pequenos times;
- usuários que necessitam organizar tarefas de forma simples e centralizada.

---

## Funcionalidades
As principais funcionalidades implementadas são:
- cadastro de usuário;
- login de usuário;
- redirecionamento para dashboard;
- criação de tarefas;
- visualização da lista de tarefas do usuário;
- autenticação e personalização por conta;
- logout.

---

## Tecnologias Utilizadas
- **Bubble** – desenvolvimento da aplicação web no-code/low-code
- **Banco de dados nativo do Bubble** – armazenamento de usuários e tarefas
- **Workflows Bubble** – automação de lógica de navegação e regras de negócio

---

## Estrutura de Dados
A aplicação foi modelada com base em duas entidades principais:

### Usuário
Campos principais:
- nome
- e-mail
- senha
- tarefas associadas

### Tarefa
Campos principais:
- título
- descrição
- usuário responsável
- data de criação (quando aplicável)

### Relacionamento
- um usuário pode possuir várias tarefas;
- cada tarefa pertence a um usuário.

---

## Principais Workflows
Os principais workflows implementados incluem:
- criação de conta;
- autenticação de usuário;
- atualização de dados do usuário;
- redirecionamento para o dashboard;
- criação de tarefas;
- exibição das tarefas do usuário logado;
- logout.

## Dificuldades Encontradas
Durante o desenvolvimento, foram identificados alguns desafios importantes:
- dificuldade na criação e organização dos workflows;
- dificuldade na configuração correta dos inputs;
- necessidade de testes constantes para validar os fluxos;
- atenção especial para garantir que os dados fossem corretamente vinculados ao usuário e exibidos no dashboard.

Essas dificuldades contribuíram para o aprendizado prático da plataforma e para a compreensão das exigências envolvidas no desenvolvimento de aplicações no-code.
